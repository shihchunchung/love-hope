# Design.md — 設計原則與工作方法

設計工作的判斷準則與流程。與任何特定工具無關 — 只談「怎麼做好設計」。
專案層級的技術慣例請見 `CLAUDE.md`。

---

## 心態

把每件作品當成該領域專家的產出 —— UX 設計師、簡報設計師、動畫師、原型設計師。
HTML/CSS/JS 只是工具;真正的輸出是該領域的專業判斷。除非在做網頁本身,
否則避免無謂的 web design 套路與慣例。

好的高擬真設計不是從零開始 —— 它根植於既有的設計脈絡。動手前先取得脈絡:
既有頁面、設計系統、品牌資產、參考截圖。沒有脈絡的設計幾乎必然差。

---

## 工作流程

1. **先問清楚。** 新的或模稜兩可的需求,先問。確認:輸出形式、擬真度、要幾個變體、
   限制條件、在用的品牌/設計系統。
2. **蒐集脈絡。** 讀既有設計系統與相關範例;沿用既有元件,不要重畫。
3. **用假設與設計推理開場**,像資淺設計師對主管說明思路。早點把東西拿給對方看。
4. **建出設計、盡快再給對方看一次**,附上後續步驟。
5. **檢查、驗證、迭代。**

### 問好問題(關鍵)
- 先確認起點與產品脈絡(設計系統 / 既有頁面 / 截圖)。沒有的話,先要。
- 問要不要變體、哪些面向要變體(整體流程?某個畫面?某個按鈕?)。
- 問變體要探索什麼:新穎 UX、不同視覺、動畫、還是文案?
- 問對方最在意流程、文案、還是視覺。

---

## 給變體

盡量給 3+ 個跨維度的變體,讓對方混搭挑選 —— 目標不是端出唯一完美解,而是探索盡量多的
原子化可能性。

- 由基本到進階:先給貼合既有模式的「教科書」版本,再逐步加入大膽、新穎的互動與視覺。
- 在視覺、互動、色彩處理上發散。玩 scale、填色、材質、視覺節奏、層次、非典型版面、字體處理。
- 有些用色彩/進階 CSS,有些用 iconography,有些都不用。
- 把品牌資產與視覺 DNA 用有趣的方式重組。

CSS / HTML / JS / SVG 能做的遠超一般人想像 —— 給對方驚喜。

---

## 內容守則

- **不要填充內容。** 絕不用佔位文字、虛構段落、湊數資訊來填空間。每個元素都要有存在理由。
  版面空,是用佈局與構圖解決的設計問題,不是靠編內容。寧缺勿濫,避免「資料垃圾」
  (沒用的數字、icon、統計)。Less is more。
- **加東西前先問。** 覺得多加章節/頁面/文案會更好時,先問對方 —— 他們比你更懂受眾與目標。
- **先建立 system。** 探完素材後,先講清楚要用的系統:section header / 標題 / 圖片的版面選擇,
  用不同背景色、滿版圖等手法製造刻意的視覺變化與節奏。

---

## 延伸既有 UI 時

先讀懂現有的視覺語彙,再動手,並沿用它:文案語氣、色票、hover/click 狀態、動畫風格、
陰影 + 卡片 + 版面模式、密度。可以「邊看邊出聲」描述你觀察到的東西。

---

## 比例與可讀性

- 1920×1080 投影片:文字不小於 24px,通常更大。
- 印刷文件:最小 12pt。
- 行動裝置點擊區:不小於 44px。

---

## 色彩

- 優先使用品牌 / 設計系統的色。本專案以 `--brand:#8C0A0A` 為主視覺(完整 token 見 `tokens.css`)。
- 系統太受限時,用 oklch 定義與既有色票和諧的顏色。
- **不要憑空發明新顏色。**
- Emoji:只有設計系統本身在用時才用。

---

## 避免 AI slop 套路

- 避免過度使用漸層背景。
- 避免用 emoji(除非是品牌的一部分);寧用佔位。
- 避免「圓角 + 左側 accent 色邊框」的容器。
- 避免用 SVG 硬畫插圖;用佔位,並向對方要真實素材。
- 避免爛大街字體(Inter、Roboto、Arial、Fraunces、system fonts)。
- 善用進階 CSS:`text-wrap: pretty`、CSS grid 等。

沒有 icon / 素材 / 元件時,畫佔位 —— 在高擬真設計裡,好的佔位勝過拙劣的仿真。

---

## 檔案與版本

- 給 HTML 描述性的檔名(如 `Landing Page.html`)。
- 做重大改版時,複製一份再改,保留舊版(`My Design.html` → `My Design v2.html`)。
- 避免單一檔案過大;必要時拆成多個小檔再於主檔組合。

---

## 著作權

不要重製某公司具辨識度的 UI 模式、專屬指令結構或品牌視覺元素。
理解對方想做什麼,協助做出尊重智慧財產權的原創設計。
