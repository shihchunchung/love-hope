§ 02

審計四準則

How I judge each spacing value — not by ratio, but by purpose.

本次審計不採「全部按比例放大」。每一個 spacing 都要回答四個問題,再決定動不動、怎麼動。

01

本意是什麼?

每個 padding / margin 都有功能 —— 是「分隔內容」、「強調結構」、「容器呼吸」或「視覺停頓」?分清楚才能判斷字級升級後本意是否還達成。

02

字級升級後本意還達成嗎?

字大了會壓縮 spacing 的「相對」呼吸感。原本 1.5x 行高的留白變相對小 —— 但有些 spacing 因為本來就過大,字升級反而剛好對齊。逐項看,別預設要加。

03

同源還是散亂?

22 / 24 / 28 三個值在不同卡上隨機出現,代表沒有共識。本次審計傾向把「同類功能的 spacing」收斂到同一值 —— 但不引入 token 系統(基金會規模不需要)。

04

編輯密度 vs 設計留白

編輯型網站不靠 white space 取勝。NYT、報導者的卡是緊的。字級拉大 ≠ 邊距也要拉大。中老年讀者要的是「看得清楚」,不是「空之又空」。寧可保留扎實密度。

06

中文身上的 letter-spacing Latin-borrowed letter-spacing on Chinese text

必動
問題核心

上一輪已把中文大標的 letter-spacing 從 2–3px 收到 1–1.5px。但內文、導覽、footer、按鈕的中文身上,仍有 .5–3px 的 letter-spacing —— 這是「給拉丁字 / 中英混排」的設計慣性,中文獨自承擔反而變散。

§ 06

刻意不動 (GREEN)

Looked at, decided to keep. Editorial density is intentional.

以下 spacing 仔細看過,刻意保留 —— 字級升級後仍合理,或調整反而破壞「編輯密度」感。逐一列舉,讓你看出我「沒漏掉」哪些。

  • section.section padding: 80px 0 section 之間 80px 上下 = 4.4× 內文,落在編輯型網站的典型 64–96px 區間,不該再加。
  • .donate-band padding: 80px 0 跟主 section 同節奏,合理。
  • .banner-slider padding: 32px 0 0 banner 與 header 之間留 32px,字升級後仍夠。
  • .donate-card padding: 64px 主視覺面板,字升級後 h2 48px,padding 64px 仍有 16px 緩衝。再加會稀釋密度。
  • .spot-text padding: 48px 56px 本就為大字級設計的 panel,字升級後仍對。
  • .article-grid gap: 56px row × 32px col row 56 給 9:5 大圖呼吸,col 32 維持卡片密度。報導型網站典型。
  • .actions-grid gap: 1px 用 line color 模擬 divider 的特殊結構,不是 spacing,不動。
  • .unit-row gap: 64px (桌機) 故事列圖文之間 64px,字升級後仍是清晰分隔。
  • 所有 container max-width 1320 / 760 / 680 / 640 / 480 各有功能,字升級後欄寬比例仍在中文舒適區。
  • 所有 mobile override padding 已是最小可行,字升級後仍能容納內容,再縮會貼邊,再放會變相對小。Sweet spot。
  • .app-header .row padding: 18px 40px header 高度由 logo 撐,字升級不影響。
  • .app-footer padding: 64px 40px 36px 外緣 padding 仍能容納升級後的內容群高度。
  • eyebrow / Serif label 的 letter-spacing 2.5–3.5px 對拉丁 uppercase 必要,中文部分共用,但因為字數短(2–6 字)影響有限,保留。

(尚有 10+ 處 spacing 用了「對的初始值」,字升級後仍對,不另列。)

§ 08

下一步討論

Questions before we ship.

這份審計刻意「保守」 —— 字級升級後我傾向動小、看效果、再決定要不要動多。落地前有幾個方向要對齊。

01

RED + AMBER 一起出,還是只先做 RED?

RED 10 處是「字升後明顯失衡」必動。AMBER 6 處是「動了會更好,但不動也不壞」。建議:RED 全做,AMBER 看完 RED 效果再決定。如果想一次到位,16 處一起出也可。

02

line-height 收斂要做到多徹底?

把 14 個值收到 6 個是大改動。徹底做 = 15+ 個選擇器逐個改。輕度做 = 只動有「明顯太鬆」的(例如 line-height:2.05 → 1.85)。前者更乾淨,後者風險低。傾向徹底做,因為 14 種太多。

03

中文 letter-spacing 歸零會不會「太密」?

不會。中文字方塊本身就有等寬距離,letter-spacing 加在中文上等於「字塊之間又多一個縫」,讓字看起來散、不實心。歸零是回到中文「該長的樣子」 —— NYT 中文版、報導者、端傳媒中文內文都是 ls 0。

04

要不要趁這次也檢查色彩 / 對比?

中老年友善的下一塊是「對比度」。muted 色 #9A9A9A 配白底是 2.8:1,低於 WCAG AA 4.5:1。如果同意,下一份審計可以做色彩。本次不涉及。

05

導入方式:全站一次換 vs 分檔測試?

跟 typography 一樣建議三檔同時換。基金會官網規模小,「混搭期」反而怪。週末完成。
